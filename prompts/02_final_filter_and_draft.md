# Prompt 模板 02：Phase 2-B 终筛 + 逐条写开场白

> 此 Prompt 用于让 LLM 对 `enrich.js` 拉取的 JD 详情做终筛，并为通过的岗位逐条撰写定制开场白。
> 输入：`data/enriched_latest.jsonl` 或 `enriched_latest.md`
> 输出：`data/approved.json`（含 `opening` 字段的数组）

---

## 使用方式

1. 打开 `data/enriched_latest.md`，复制全文
2. 将下方 Prompt 发送给 LLM，粘贴内容
3. 让 LLM 输出 JSON 数组，保存为 `data/approved.json`
4. 运行 `node send.js`（先 dry-run，再 `--send`）

---

## Prompt 模板

```
你是一个求职助手，帮我做 BOSS 直聘岗位的最终筛选，并为通过的岗位写定制开场白。

## 我的真实经历（只能用这些，不能虚构）
[粘贴 profile.local.md 里允许用于开场白的真实事实；没有写在里面的经历一律不能用]

示例（替换成你自己的真实经历，不要照搬）：
- [实习/工作经历：做过的具体事，如 项目支持：需求记录、物料跟进、客户接待、资料归档]
- [校园/社会经历：如 办公室助理，10+ 场活动资料准备、100+ 份文件归档]
- [可量化的事：如 日均 1000+ 物资调度，Excel 数据核查零差错]
- 技能：[如 Office / Excel / PS / 驾照 / 普通话等级]
- 学历：[你的学历与专业]

## 我的保底线
[填写已有 offer / 当前工作 / 最低可接受条件。通过岗位必须跑赢它；只有明确写入 profile.local.md 的长期成长线可以例外。]

## 终筛规则（读 JD 详情后判断）

### 直接剔除（任意命中即丢弃）：
1. **培训贷/刷单**：JD 含"免费学习""日入200-600""每天5小时空闲""轻松赚钱"
2. **产线工伪装**：含"两班制""每天10小时""每月26天""夜班补贴"（文职岗出现这些 = 产线）
3. **形象/外貌门槛**：含"身高162以上""全身生活照""颜值""形象气质佳"（非服务业的话是信号）
4. **保险增员**：保险公司 + 含"整理老客户资料""吸引筛选人才""发展团队"
5. **隐藏销售**：非销售岗名 + JD 含"邀约客户""签单""转化率""获客""线索"
6. **跑不赢保底线**：薪资、作息、社保、通勤、公司稳定性明显不如我的保底线，且不属于长期成长线

### 通过后写开场白

对通过终筛的岗位，逐条写 45-120 字的定制开场白，要求：

**内容要求：**
- 把我的真实经历翻译成这个岗位的语言，不要照抄 JD 关键词
- 只写我真实做过的事，不能虚构
- 一条只讲一个核心印象（比如"做过大量数据核查"或"跟过物料落地流程"）
- 前 20 个字要贴住 JD 的具体动作，不要以"我是..."开头

**风格要求（非常重要）：**
- 像真人随口发的微信消息，不像述职报告
- 不写"我理解这个岗位""我对贵公司很感兴趣""认真负责踏实肯学"这类废话
- 不要名词堆砌，不要破折号长句，不要一句话塞太多动作
- 写完默念一遍，听起来像真人打的就行，太顺太满就是 AI 感
- 结尾用低压行动信号，例如"简历已更新，方便时可以看一下"或"随时可以面试，能尽快到岗"

## 输出格式

输出 JSON 数组，每项包含以下字段（直接从输入数据复制 ID 字段）：
{
  "encryptBossId": "（从输入复制）",
  "securityId": "（从输入复制）",
  "encryptJobId": "（从输入复制）",
  "lid": "（从输入复制）",
  "jobName": "岗位名",
  "brandName": "公司名",
  "bossName": "HR姓名",
  "postDescription": "从输入复制 JD 全文",
  "opening": "你写的开场白",
  "judge": {
    "evidence": "从 JD 原文复制一句最关键证据",
    "companyAssessment": "公司是否正常、是否有销售/金融/外包客服风险",
    "riskDecision": "命中风险词时说明为什么仍可投；无风险也写明",
    "baselineDecision": "为什么跑赢我的保底线，或为什么属于长期成长线例外",
    "reason": "一句话推荐理由"
  }
}

## 岗位列表（含 JD 详情）

[粘贴 enriched_latest.jsonl 或 enriched_latest.md 的内容]
```

## 开场白质量检查清单

写完后对照检查，有任意一条就重写：

- [ ] 是否用了"我理解""对贵公司很感兴趣""认真负责""踏实肯学"这类废话
- [ ] 一句话里是否塞了 3 个以上动词（"做过/负责/参与/跟进/维护..."）
- [ ] 是否用了破折号长定语
- [ ] 是否把 JD 关键词原样串起来（比如"熟悉行政文员日常工作流程"）
- [ ] 是否虚构了你没做过的事
- [ ] 是否缺少 JD 原文证据和保底线判断
- [ ] 口语化程度：朗读一遍，是否像真人随口说的

---

## 注意事项

- 每条开场白只为这一个岗位定制，**禁止套模板**
- `opening` 字段是 send.js 发送的内容，写好了才能真发
- 完成后运行：`node send.js`（先 dry-run 验证，再加 `--send` 真发）
